## 10. 事件类型汇总

| 常量 | 值 | 描述 |
|-----|----|----|
| `EventTypeSagaStart` | `start-event` | 启动Saga |
| `EventTypeStepExecute` | `saga.step.execute` | 执行步骤 |
| `EventTypeStepSuccess` | `event-success` | 步骤成功 |
| `EventTypeStepFailed` | `event-failed` | 步骤失败 |
| `EventTypeStepCompensate` | `saga.step.compensate` | 补偿步骤 |
| `EventTypeStepRecoverySuccess` | `event-recover-success` | 补偿成功 |
| `EventTypeStepRecoveryFail` | `event-recover-fail` | 补偿失败 |
| `EventTypeSagaInitiated` | `saga.initiated` | Saga已创建 |
| `EventTypeSagaCompleted` | `saga.completed` | Saga完成 |
| `EventTypeSagaCompensated` | `saga.compensated` | Saga已补偿 |

## 11. 核心能力总结

1. **执行模式**: 支持串行 (sequential) 和并行 (parallel) 两种执行模式
2. **容错机制**: 步骤重试 + 补偿回滚 + 死信队列
3. **可配置性**: YAML 模板定义事务流程
4. **并发安全**: 多层锁保护 + 状态机验证
5. **可观测性**: 完整的事件生命周期追踪
6. **超时检查**: 每秒检查运行中的 Saga 是否超时
